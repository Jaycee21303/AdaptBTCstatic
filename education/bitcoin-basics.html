Bitcoin Basics

Start with the fundamentals: what Bitcoin is, why it was created, and how the network stays secure.

← Back to Education Hub

Last updated: January 13, 2026

What you’ll learn

  • Why Bitcoin was designed as an alternative to traditional money.
  • How blocks, miners, and nodes coordinate to secure the network.
  • What it means to “own” bitcoin and why keys matter.
  • Where transactions, fees, and wallets fit into the system.

Why Bitcoin exists

Bitcoin was created to allow value transfer without relying on a central authority. It uses a public ledger so anyone can verify the supply and transaction history, while a fixed issuance schedule ensures the supply is predictable and capped.

Instead of trusting an intermediary to keep records honest, the system relies on economic incentives and open verification so that anyone can audit the rules.

Note

Bitcoin is open-source software. Anyone can run a node, verify the rules, and choose which software version they follow.

How the network works

Miners gather transactions into blocks and compete to add them to the chain using proof-of-work. Nodes independently verify the blocks, ensuring that transactions follow the consensus rules.

Consensus at a glance

Nodes enforce the rules, miners provide security through computation, and users control their own keys. This separation keeps the system resilient because no single party controls all pieces.

Ownership and wallets

Bitcoin ownership is tied to private keys. A wallet is a tool that manages those keys and signs transactions. If you control the keys, you control the bitcoin. If a custodian controls them, they do.

Different wallets offer different trade-offs between convenience and security, which you’ll explore in the custody lessons.

Key takeaways

  • Bitcoin replaces centralized trust with open verification.
  • Nodes verify the rules; miners secure the chain.
  • Keys are the core of ownership—protect them first.
