Lightning Network Basics

Discover how Lightning enables fast, low-fee payments while keeping Bitcoin’s base layer secure.

← Back to Education Hub

Last updated: January 13, 2026

What you’ll learn

  • How payment channels move transactions off-chain.
  • Why routing and liquidity matter for Lightning payments.
  • What channel management looks like in practice.
  • How Lightning connects back to on-chain settlements.

Payment channels in a nutshell

Lightning uses two-party payment channels that lock funds on-chain and allow many instant payments off-chain. Only the opening and closing transactions touch the base layer.

Routing and liquidity

Lightning payments often hop through multiple channels. Nodes need balanced liquidity to forward payments, so channel management is part of running a reliable Lightning setup.

Tip

Wallets can abstract Lightning complexity, but teams running their own nodes should plan for monitoring and liquidity management.

Settlement and security

Lightning relies on Bitcoin’s base layer for final settlement. If a dispute occurs, the protocol allows participants to close channels on-chain and enforce the latest valid balance.

Key takeaways

  • Lightning enables fast payments without changing Bitcoin’s base rules.
  • Liquidity and routing are key to successful payments.
  • On-chain settlement provides security backstop.
